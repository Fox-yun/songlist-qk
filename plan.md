# songlist

计划采用 `SvelteKit + TypeScript + Tailwind CSS + Supabase(Postgres)` 构建一个轻量但现代的全栈直播歌单站，支持歌单展示、搜索、观众提交歌曲愿望单，以及主播后台管理。

## 初步方案

- 做成**轻量全栈单体项目**：前台页面、接口逻辑和后台管理放在一个项目里，降低复杂度。
- 使用 `SvelteKit`，原因是它比传统前后端分离更轻、更现代，开发体验好，适合快速做 MVP。
- 使用 `Supabase` 承担数据库、鉴权和基础后端能力，能明显减少后端样板代码。

## 范围

### 1. 观众端
- 主播信息展示区
- 可搜索歌单列表
- 歌曲分类/标签筛选
- 歌曲愿望单提交入口
- 提交结果反馈

### 2. 主播后台
- 管理员登录
- 歌曲增删改查
- 歌曲愿望单查看与状态更新（已加入日程、已学、已拒绝）
- 基础概览信息（歌曲总数、待处理愿望数）


## 产品决策

- **愿望单与正式歌单分离**：观众提交的愿望不要求必须命中现有歌单，便于收集“还不会唱但想学”的歌。
- **观众无需登录**：降低直播场景下的使用门槛。
- **后台仅服务主播本人**：MVP 先做单管理员模式。
- **搜索优先前端即时搜索**：歌单规模在早期可控，体验更轻量。
- **愿望单优先异步处理**：MVP 不做实时排队系统，先聚焦“收集 - 处理 - 状态同步”。

## 非目标与后续扩展

- MVP 暂不做观众账号体系
- 暂不做实时刷新和弹幕联动
- 暂不做复杂推荐、热度排序与审核流
- 后续可扩展二维码分享、愿望单去重增强、演唱历史
